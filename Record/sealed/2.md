Yes, **sealed can use `extends`, `implements`, and `permits` together** ✅
But **there are clear rules** about *how* and *when*.

I’ll explain **slowly, clearly, with code, comments, diagrams-in-words, output, and interview traps**.

---

## 1️⃣ Short Direct Answer (Interview Ready)

👉 **YES**, a sealed class **can**:

* `extend` a class
* `implement` one or more interfaces
* `permit` specific subclasses

👉 **All together in the same declaration**.

---

## 2️⃣ General Syntax (IMPORTANT)

```java
public sealed class Child
        extends Parent
        implements Interface1, Interface2
        permits Sub1, Sub2 {
}
```

✔ This is **100% valid Java (17+)**

---

## 3️⃣ Complete Working Example (Extend + Implement + Permit)

### Step 1: Normal Parent Class

```java
public class BaseEntity {
    protected Long id;
}
```

---

### Step 2: Interface

```java
public interface Auditable {
    void audit();
}
```

---

### Step 3: Sealed Class using ALL THREE

```java
public sealed class User
        extends BaseEntity
        implements Auditable
        permits AdminUser, CustomerUser {

    protected String name;

    public User(Long id, String name) {
        this.id = id;
        this.name = name;
    }
}
```

✔ `extends` → BaseEntity
✔ `implements` → Auditable
✔ `permits` → AdminUser, CustomerUser

---

### Step 4: Final Subclasses

```java
public final class AdminUser extends User {

    public AdminUser(Long id, String name) {
        super(id, name);
    }

    @Override
    public void audit() {
        System.out.println("Auditing admin user");
    }
}
```

```java
public final class CustomerUser extends User {

    public CustomerUser(Long id, String name) {
        super(id, name);
    }

    @Override
    public void audit() {
        System.out.println("Auditing customer user");
    }
}
```

---

### Step 5: Test Class

```java
public class Test {
    public static void main(String[] args) {

        User u1 = new AdminUser(1L, "Saurabh");
        User u2 = new CustomerUser(2L, "Rahul");

        u1.audit();
        u2.audit();
    }
}
```

---

### Output

```text
Auditing admin user
Auditing customer user
```

## 4️⃣ Sealed Interface using `extends` + `permits`

Yes, **sealed interface can also use both**.

```java
public sealed interface Payment
        extends Auditable
        permits UpiPayment, CardPayment {
}
```

```java
public final class UpiPayment implements Payment {
    @Override
    public void audit() {
        System.out.println("UPI audit");
    }
}
```

## 5️⃣ Important Rules (INTERVIEW TRAPS ⚠️)

### Rule 1️⃣: Order matters

Correct order: permits should be the last

```java
sealed class X
    extends A
    implements B
    permits C, D {}
```

❌ Wrong order:

```java
sealed class X permits C extends A {} // ❌ invalid
```

---

### Rule 2️⃣: You can extend ONLY ONE class

(Java rule, not sealed rule)

```java
sealed class A extends B, C permits D {} // ❌
```

---

### Rule 3️⃣: Permitted classes must follow sealing rules

Each permitted class **must be**:

* `final`
* `sealed`
* `non-sealed`

---

### Rule 4️⃣: Permitted class must directly extend sealed class

❌ This is illegal:

```java
sealed class A permits C {}
class B extends A {}
final class C extends B {} // ❌
```

## 6️⃣ Sealed + Non-Sealed Mix Example (Advanced)

```java
public sealed class Account
        permits SavingsAccount, LoanAccount {
}
```

```java
public non-sealed class SavingsAccount extends Account {
}
```

```java
public class SeniorCitizenSavings extends SavingsAccount {
}
```

✔ Allowed because `SavingsAccount` is `non-sealed`

## 7️⃣ Tricky Interview Questions (WITH ANSWERS)

### Q1️⃣ Can sealed class implement multiple interfaces?

✅ **Yes**

---

### Q2️⃣ Can sealed interface extend another interface?

✅ **Yes**

---

### Q3️⃣ Can sealed class extend another sealed class?

✅ **Yes**

---

### Q4️⃣ Can a permitted class implement extra interfaces?

✅ **Yes**

---

### Q5️⃣ Can abstract class be sealed?

✅ **Yes** (very common)

---

### Q6️⃣ Is `permits` mandatory?

❌ No, if subclasses are in the **same file**

---

### Q7️⃣ Can sealed class be generic?

✅ **Yes**

```java
public sealed class Box<T> permits StringBox {}
```

---

### Q8️⃣ Can Spring/JPA entities be sealed?

⚠️ **Not recommended**
Hibernate creates proxies → sealed breaks this

---

## 8️⃣ streams.One-Line Interview Answer 💯

> **Yes**, a sealed class can **extend a class**, **implement interfaces**, and **use permits together**, as long as Java inheritance rules and sealed-class constraints are followed.
