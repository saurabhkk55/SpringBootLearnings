## 1️⃣ What is ModelMapper?

**ModelMapper** is a Java library used to **map one Java object to another Java object** automatically.

👉 Typical use case in Spring Boot:

- **Entity ↔ DTO**
- - **DTO ↔ Response Object**
- - **Request DTO ↔ Entity**

It reduces **boilerplate setters/getters code**.

---

## 2️⃣ Do we use ModelMapper for **JSON ↔ Java object** conversion?

### ❌ NO (Not directly)

**JSON ↔ Java Object conversion is done by Jackson**, not ModelMapper.

### Flow in Spring Boot:

```
JSON  → (Jackson) → RequestDTO
RequestDTO → (ModelMapper) → Entity
```

### Example:

```json
{
  "name": "Saurabh",
  "age": 25
}
```

- Jackson converts JSON → `UserRequestDTO`
- ModelMapper converts `UserRequestDTO` → `UserEntity`

📌 **ModelMapper never reads JSON directly**

---

## 3️⃣ Do we use ModelMapper for **Java object ↔ Java object** conversion?

### ✅ YES (This is its actual purpose)

This is **exactly what ModelMapper is designed for**.

### Example:

```java
UserEntity entity;
UserDTO dto = modelMapper.map(entity, UserDTO.class);
```

### Common conversions:

| From         | To           |
| ------------ | ------------ |
| Entity       | DTO          |
| DTO          | Entity       |
| RequestDTO   | Domain Model |
| Domain Model | ResponseDTO  |

---

## 4️⃣ Why use ModelMapper instead of manual mapping?

### ❌ Manual Mapping (Boilerplate)

```java
dto.setName(entity.getName());
dto.setEmail(entity.getEmail());
dto.setAge(entity.getAge());
```

### ✅ ModelMapper

```java
modelMapper.map(entity, UserDTO.class);
```

### Benefits:

- Less code
- Cleaner service layer
- Easy refactor
- Supports nested mapping

---

## 5️⃣ Which **annotations** are used with ModelMapper?

- ⚠️ **Important clarification**
- 👉 ModelMapper **does NOT provide its own annotations** like JPA or Jackson.

It **works with standard Java + Jackson annotations**.

---

### 🔹 Common annotations used *along with* ModelMapper

#### ✅ `@Bean`

To register ModelMapper in Spring

```java
@Bean
public ModelMapper modelMapper() {
    return new ModelMapper();
}
```

---

## ModelMapper & Jackson – Clear Separation of Responsibilities

**ModelMapper does not have its own annotations**, unlike Jackson which provides annotations such as:

- * `@JsonProperty`
- * `@JsonIgnore`
- * `@JsonInclude`

### Role of Jackson

Jackson is responsible for:

- * Converting **JSON → Java Object**
- * Converting **Java Object → JSON**
- * Applying JSON-related annotations (`@JsonProperty`, `@JsonIgnore`, `@JsonInclude`, etc) during this process

Example:

```json
{
  "name": "Saurabh",
  "age": 25
}
```

- ➡ Jackson converts JSON into `UserRequestDTO`
- ➡ While doing so, Jackson uses annotations like `@JsonProperty`, `@JsonIgnore`, etc.

---

### Role of ModelMapper

Once Jackson has converted JSON into a Java object (`UserRequestDTO`), **ModelMapper is used to map that Java object to another Java object**, for example:

```java
UserEntity userEntity = modelMapper.map(userRequestDTO, UserEntity.class);
```

- 📌 **ModelMapper never handles JSON directly**
- 📌 It only works with **Java object ↔ Java object conversion**

---

### Spring Boot Flow (Correct Mental Model)

```
JSON
  ↓ (Jackson + Jackson annotations)
RequestDTO
  ↓ (ModelMapper)
Entity
```

---

## 🎯 Interview streams.One-Liner (Very Important)

> **Jackson handles JSON serialization/deserialization, while ModelMapper handles Java object-to-object mapping such as DTO to Entity conversion.**
