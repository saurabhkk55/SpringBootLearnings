# 🔹 What is MapStruct?

**MapStruct** is a **Java annotation-based code generator** used for **mapping one Java object to another** (mainly **DTO ↔ Entity**).

➡️ It generates **compile-time mapping code** (not runtime reflection).

---

# 🔹 Why do we use MapStruct?

### ❌ Problems without MapStruct

Manual mapping is:

```java
UserDTO dto = new UserDTO();
dto.setId(user.getId());
dto.setName(user.getName());
dto.setEmail(user.getEmail());
```

Issues:

* ❌ Too much **boilerplate code**
* ❌ Error-prone (missing fields)
* ❌ Hard to maintain when fields change
* ❌ Slower if reflection-based mappers are used

---

### ✅ Benefits of MapStruct

| Benefit             | Explanation                            |
| ------------------- | -------------------------------------- |
| 🚀 Fast             | No reflection → pure Java code         |
| 🛡️ Type-safe       | Compile-time errors if fields mismatch |
| 🧹 Clean code       | No manual setters                      |
| 🔧 Easy maintenance | Field change → compiler tells you      |
| 🧪 Test-friendly    | Simple Java methods                    |

👉 **Best for Spring Boot DTO ↔ Entity mapping**

---

# 🔹 How MapStruct Works (Internally)

1. You define a **Mapper interface**
2. MapStruct **generates implementation at compile time**
3. Generated class uses **plain Java setter/getter calls**

📌 Example:

```java
UserMapper mapper = Mappers.getMapper(UserMapper.class);
```

---

# 🔹 Commonly Used MapStruct Annotations

## 1️⃣ `@Mapper`

Marks an interface as a MapStruct mapper.

```java
@Mapper
public interface UserMapper {
}
```

For Spring Boot:

```java
@Mapper(componentModel = "spring")
```

---

## 2️⃣ `@Mapping`

Maps fields when:

* Field names are different
* Custom transformation needed

```java
@Mapping(source = "emailAddress", target = "email")
```

---

## 3️⃣ `@Mappings` (Optional in new versions)

Used to group multiple mappings (older style).

```java
@Mappings({
   @Mapping(source = "firstName", target = "name"),
   @Mapping(source = "emailId", target = "email")
})
```

---

## 4️⃣ `@MappingTarget`

Used for **updating existing objects** (PATCH use-case).

```java
void updateUser(UserDTO dto, @MappingTarget User entity);
```

---

## 5️⃣ `@InheritInverseConfiguration`

Auto-reverse mapping.

```java
@InheritInverseConfiguration
User dtoToEntity(UserDTO dto);
```

---

## 6️⃣ `@Named`

Used with `qualifiedByName` for custom methods.

```java
@Named("toUpper")
```

---

## 7️⃣ `@AfterMapping` / `@BeforeMapping`

For extra logic.

```java
@AfterMapping
default void after(UserDTO dto, @MappingTarget User user) {
    user.setUpdatedAt(LocalDateTime.now());
}
```

# 🔹 Complete Working Example (Proper Syntax)

---

## 🔸 Entity

```java
@Entity
public class User {

    private Long id;
    private String name;
    private String email;
    private int age;
}
```

---

## 🔸 DTO

```java
public class UserDTO {

    private Long userId;
    private String fullName;
    private String email;
}
```

---

## 🔸 Mapper Interface

```java
@Mapper(componentModel = "spring")
public interface UserMapper {

    @Mapping(source = "id", target = "userId")
    @Mapping(source = "name", target = "fullName")
    UserDTO toDto(User user);

    @InheritInverseConfiguration
    User toEntity(UserDTO dto);
}
```

📌 MapStruct auto-generates:

```java
UserMapperImpl
```

---

## 🔸 Usage in Service Layer

```java
@Service
public class UserService {

    private final UserMapper userMapper;

    public UserService(UserMapper userMapper) {
        this.userMapper = userMapper;
    }

    public UserDTO getUserDto(User user) {
        return userMapper.toDto(user);
    }
}
```

---

# 🔹 PATCH Update Example (Very Important for Interviews)

```java
@Mapper(componentModel = "spring")
public interface UserMapper {

    void updateUserFromDto(UserDTO dto, @MappingTarget User entity);
}
```

✔ Updates only provided fields
✔ Existing object remains intact

# 🔹 MapStruct vs ModelMapper (Interview Question)

| MapStruct               | ModelMapper     |
| ----------------------- | --------------- |
| Compile-time            | Runtime         |
| Faster                  | Slower          |
| Type-safe               | Not fully       |
| No reflection           | Uses reflection |
| Preferred in production | Mostly avoided  |

---

# 🔹 When NOT to Use MapStruct?

❌ Very dynamic mappings
❌ Mapping depends heavily on runtime logic

---

# 🔹 streams.One-Line Interview Answer 💡

> **MapStruct is a compile-time code generation library used to map DTOs and entities efficiently without reflection, providing high performance, type safety, and clean code.**
